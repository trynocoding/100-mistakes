# 八进制整数陷阱

## 传统八进制写法的问题

在 Go 1.13 之前，八进制整数使用 `0` 前缀表示。例如 `010` 表示八进制的 10，即十进制的 8。

这种写法容易与十进制数混淆：

```go
sum := 100 + 010 // 这里是 108，不是 110
```

`010` 看起来像十进制的 10，但实际上是八进制，导致意外的计算结果。

## Go 1.13+ 推荐的 0o 前缀写法

Go 1.13 引入了 `0o`（零和小写字母 o）前缀来明确表示八进制数，这种写法更清晰、不易混淆：

```go
sum := 100 + 0o10 // 正确：0o10 = 八进制 10 = 十进制 8，结果是 108
```

## 实际使用场景：文件权限

八进制数最常见的应用场景是文件权限。Unix/Linux 系统中，文件权限使用八进制表示：

| 权限 | 八进制 | 说明 |
|------|--------|------|
| rwxr-xr-x | 0755 | 所有者可读写执行，组和其他可读执行 |
| rw-r--r-- | 0644 | 所有者可读写，组和其他只读 |
| r-------- | 0400 | 仅所有者可读 |

```go
// 正确使用 0o 前缀表示文件权限
file, err := os.OpenFile("test.txt", os.O_CREATE|os.O_WRONLY, 0o644)

// 目录权限通常设置为 0755
os.MkdirAll("mydir", 0o755)
```

## 关键区别

| 写法 | 含义 | 十进制值 |
|------|------|----------|
| `010` | 八进制（传统写法） | 8 |
| `0o10` | 八进制（Go 1.13+ 推荐） | 8 |
| `10` | 十进制 | 10 |

## 最佳实践

1. **始终使用 `0o` 前缀**：Go 1.13+ 推荐使用 `0o` 前缀表示八进制数
2. **避免使用 `0` 前缀**：传统 `0` 前缀的八进制写法容易与十进制混淆
3. **文件权限必须显式标注**：使用 `0o` 前缀让权限设置一目了然

## 注意事项

- Go 不允许像 C 语言那样使用 `08` 或 `09` 这样的无效八进制数
- 十六进制使用 `0x` 前缀，二进制使用 `0b` 前缀（Go 1.13+）
